우여곡절 끝에 만든 Jekyll 블로그 (1) - 시작하기

프롤로그


한 달 전쯤..
한두 시간이면 만들 줄 알았던 지킬블로그를 만들다가 포기하고 오늘에서야 완성했다.

사실은 그렇게 어렵지 않은데 헤매고 있는 나같은 사람을 돕고싶다..

(macOS Mojave 기준으로 작성하여서 윈도우나 리눅스는 조금 다를 수 있다.)

0. 지킬(jekyll)블로그가 뭘까?


Jekyll 은 아주 심플하고 블로그 지향적인 정적 사이트 생성기입니다.

jekyll 공식 홈페이지에 써있는 말이다.
텍스트파일을 웹사이트에 띄울 수 있게 도와주는 것이라고 생각하면 된다.
어떻게? 정적으로!

설치하고 사용하다보면 점점 이해가 되니 간단하게 생각하는게 정신건강에 좋다.

1. 시작하기


이미 충분히 많은 블로그가 jekyll 블로그 만드는 법을 설명하고 있다.
최신의 그리고 많은 블로그를 보다보면 결국 블로그를 만들 수는 있다.
하지만 나는 제일 좋은 방법을 소개하고 싶다.

지킬 공식 홈페이지를 참고하는 것!!


지킬 공식홈페이지
지킬 공식홈페이지(한국어번역)

한국어번역 홈페이지의 경우 최신버전의 번역이 아닐 수 있기 때문에 영문판 공식홈페이지를 추천한다.
공식홈페이지의 DOCS 메뉴를 들어가서 보면 된다.
귀찮은 사람을 위해 홈페이지의 내용을 정리하여서 아래에 간략하게 써본다.

하지만 홈페이지를 보고 따라하는 것을 강력추천한다!!

2. 루비(Ruby) 개발환경 설치하기


1.command-line tools을 설치하자

xcode-select --install

2.Homebrew를 설치하자
-Homewbrew 란? macOS를 위한 패키지 관리자이다.
(다른 프로그램 편하게 받게 해주는 것이라고 생각하면 편하다.)

# Install Homebrew
/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

3.Ruby를 설치하자
-Ruby 란? 인터프리터 방식의 객체 지향 스크립트 언어이다. 맥에는 기본적으로 Ruby가 설치되어있으나, (이런저런 이유로) 최신의 Ruby를 새로 까는 것이 편하다.

brew install ruby

그 후에는 아래 명령어를 통해 .bash_profile 에 한 줄 추가해주는데

vi ~/.bash_profile

를 통해 .bash_profile 파일을 열고

export PATH=/usr/local/opt/ruby/bin:$PATH

마지막 줄에 추가해주면 된다.

3. jekyll 블로그 만들어보기



기본 준비가 끝났다.
jekyll 홈페이지 메인화면에 있는 것처럼 그냥 만들면 된다.

#bundler 와 jekyll 을 설치한다.
gem install bundler jekyll

#my-awesome-site 라는 폴더에 블로그를 만든다.
jekyll new my-awesome-site  
  
#만든 폴더로 이동한다
cd my-awesome-site  
  
#localhost에 호스팅해보자
bundle exec jekyll serve

http://127.0.0.1:4000 으로 접속해보면 아래와 같이 첫 jekyll 블로그가 완성된 것을 볼 수 있다!

혹시 아래와 같은 에러가 뜬다면

 Could not find 'bundler' (1.17.2) required by your /Users/hyunsuk/coding/test/Gemfile.lock. (Gem::GemNotFoundException)
To update to the latest version installed on your system, run `bundle update --bundler`.
To install the missing version, run `gem install bundler:1.17.2`

아래 명령을 통해 해결할 수 있다.

bundle update --bundler
gem install bundler:1.17.2

4. 배포하기


위에서 만든 블로그는 다른 사람이 접속할 수가 없다.
github pages 를 이용하면 쉽고, 무료로 블로그를 호스팅 할 수 있다.
github pages 사용법은 역시 공식홈페이지에 아주 자세하게 나와있다.

혹시 github page 기능을 먼저 써본 사람 중에서, 여러 블로그들을 돌아다니다가 궁금점이 생긴 사람이 있을 것이다.

www.github.com/repository이름 의 주소로 page가 만들어지는데
기술블로그의 주소는 깃헙아이디.github.io 인 경우가 많다.

방법은 간단한데, github 저장소 이름을

깃헙아이디.github.io

로 만들어주면 github pages 에서 깃헙아이디.github.io 주소로 접근할 수 있게 해준다.

정말 간단하게 요약하자면,

  1. 깃허브에 깃헙아이디.github.io 라는 이름의 저장소를 만들어준다.
  2. 그 저장소에 블로그를 올린다.
  3. 저장소를 github pages 기능을 이용하여 배포한다.(공식홈페이지 참고)

불친절할 수 있지만, 깃허브 이용법이나 저장소 생성, push 같은 내용은 다 알 것이라고 생각하고 생략한다. 인터넷에 찾아보면 엄청나게 자세하게 설명되어 있다..

5. 마무리


jekyll 블로그 만드는 것, 생각보다 어렵지 않다.
세 가지만 알면 된다.

  • Ruby 설치 후에 jekyll 설치하기
  • 잘 안되면 공식 홈페이지 보고 하기!!
  • 깃헙아이디.github.io 라는 이름의 repository 에 올려서 배포하기

하지만 이런 기본적인 블로그를 사용하고 싶어하는 사람은 많지 않다.
다음 글에서는 예쁜 블로그를 만들 수 있는 테마 적용법에 대해 다루겠다.
